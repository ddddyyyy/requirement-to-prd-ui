# Requirement to PRD & UI Prototype

## 功能说明

将用户需求对话转化为结构化的产品需求文档(PRD)和网页原型。

## 工作流程

本技能支持灵活的工作流程,可以从任意步骤开始:

### 核心功能

1. **完整PRD生成** - 根据用户输入，先进行需求澄清，然后生成包含需求背景、功能描述、用户故事、验收标准、原型图、UI设计稿等的完整PRD文档
2. **文档自检** - 自动检测文档中的冲突和不一致问题
3. **网页原型生成** - 根据UI设计稿生成可交互的HTML原型

## 使用方法

### 开始使用

直接告诉我你的产品需求，比如：
```
我想做一个任务管理应用
```

### 可用命令

- `/prd-generate` - 生成PRD文档
- `/prd-check` - 检查现有PRD文档的冲突
- `/prototype-generate` - 根据PRD生成网页原型
- `/full-process` - 执行完整流程，从需求澄清开始执行，每一步向用户确认没问题之后，就将上一步的输出作为下一步的输入，自动进行下一步

## 工作流程

1. **PRD生成** - 生成完整的产品需求文档
2. **文档自检** - 检查术语一致性、逻辑一致性、UI和需求是否一致、依赖关系等
3. **冲突确认** - 如果发现问题，会列出让你确认，根据用户确认的结果，如果用户要求修改，则对文档进行重新修改，并重复检查，直到没有冲突为止
4. **原型生成** - 根据前面步骤生成的PRD文档里的UI设计稿，或者用户提供的UI设计稿，生成可交互的网页原型

## 输出文件

- 完整的PRD文档（MD文档）
- 网页原型文件（可在浏览器直接打开的HTML文件）
